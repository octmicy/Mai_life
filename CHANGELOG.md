# 更新日志

## [1.8.0] - 2026-07-16

- 旧版英文指令继续作为隐藏兼容入口，避免已有脚本和使用习惯在升级后立即失效。
- `/麦麦管理` 支持“概览、用户、群聊、日期、来源、书柜、统计、主动”中文范围，并统一映射到内部管理服务。
- 插件、配置与菜单显示版本升级到 `1.8.0`，SQLite Schema 仍为 v9。

- 保留并校正原有注释与类型注解；本版本不改变业务条件、数据库 Schema 或默认配置行为。
- 插件、配置与菜单显示版本升级到 `1.7.2`，SQLite Schema 仍为 v9。

## [1.7.1] - 2026-07-14

### 指令菜单

- 新增 `/mai` 主命令，`/mai` 与 `/mai_help` 现在返回本地 Pillow 渲染的完整指令菜单。
- 菜单采用参考图的冰蓝、浅紫、半透明白色玻璃面板和红黄绿窗口圆点，并缓存相同内容以降低重复渲染开销。
- 命令发送前按真实 QQ 用户重新解析当前会话，兼容 SnowLuma、NapCat 经过 MaiBot Host 暴露的标准流能力。

- Manifest 增加 `send.image` 与 `chat.get_all_streams` 能力，配置与插件版本升级到 `1.7.1`；SQLite Schema 仍为 v9。

## [1.7.0] - 2026-07-14

### 身份与消息

- 用户、管理员、关系权限和群目标统一只按真实 QQ 号判断；昵称与群名称改为 Host 自动展示快照。
- 群聊新增独立防抖开关和等待参数，按“平台 + 群号 + 发送者 QQ”隔离。
- `/mai_relay` 改为按群 QQ 号执行简单转述，删除群别名、群友关系解析和 `at` 消息段构造。
- 主动私聊删除全局每日额度，只保留每个用户明确的 `0～20` 上限。

### 联网搜索

- 新闻与主动搜索统一使用有序 API 服务链，内置博查、Tavily、You.com，并支持 OpenAI Responses/Chat 兼容联网中转。
- 每个服务支持多个 Key；实现鉴权停用、429/额度冷却、主 Key 恢复、服务故障降级和单次 12 请求硬上限。
- SQLite 只保存 Key 指纹和健康状态；原始 Key 不进入日志、Prompt、统计或管理 API。
- 每日额度按逻辑搜索尝试计数，空结果不惩罚 Key，但不会形成分钟级重复请求；无可用服务时跳过查询规划模型。
- 新闻改为兴趣轮换 API 搜索，最多保留 5 条结果、读取前 3 篇正文，并批量执行一次整理和一次自我关联。
- 删除 RSS、Atom、B站插件 API、SearXNG 和通用 JSON 映射入口。

### 数据与兼容

- 删除技能成长配置、路由、命令、Prompt、结算逻辑和持久化数据。
- 配置与 manifest 升级到 `1.7.0`，SQLite Schema 升级到 v9。
- v8 升级会删除技能和手动别名数据，并把旧主动额度 `-1` 转为主人 `2`、朋友 `1`。
- `/mai_admin relations` 替换为 `/mai_admin groups`；Token 统计单独展示搜索 API 请求，不伪装成 Token。
- 扩展 SnowLuma/NapCat 的群防抖、撤回关联、纯文本转述和联网回归测试。

## [1.6.0] - 2026-07-14

### 新增

- SnowLuma 0.8.2 与 NapCat Adapter 1.3.1 共用的 OneBot 好友/群消息撤回通知归一化。
- 防抖等待、Replyer 输出和每个发送分段前的撤回取消链路；热重载后仍可读取短期持久墓碑。
- 可选本人私聊撤回摘要，以及 `/mai_recalled` 和自然语言查询背景。

### 隐私与边界

- 撤回只有一个总开关，同时覆盖私聊和群聊；摘要缓存默认关闭。
- 摘要只允许原发送者本人在配置私聊中查询，群聊内容永不缓存，图片二进制永不入库。
- 已成功发送的 Bot 消息不自动撤回，只取消尚未发送的回复和剩余分段。
- 不加入违禁词扫描或基于撤回内容的自动平台撤回。
- 撤回会清理关联的互动、休息积压、图片摘要、日期候选、群摘要和未发送转述候选。
- 配置 Schema 与插件版本升级为 `1.6.0`，SQLite Schema 升级为 v8。

## [1.5.1] - 2026-07-13

### 新增

- WebUI 鉴权首页中的“麦麦生活管理”声明式卡片，可直达插件配置。
- `/mai_admin` 聚合管理命令，覆盖用户、关系、日期、来源、书柜、Token 和主动候选。
- `public=False` 的 `admin_snapshot` 私有 API，为未来 Host 鉴权桥保留结构化数据接口。

### 安全与兼容

- 不修改 Dashboard、不运行自建 HTTP 服务，也不开放无鉴权私密 API。
- 管理摘要移除聊天原文、书柜正文、流 ID、URL 凭据、查询参数和 API Key。
- Planner/Replyer 后处理严格限制到配置私聊和本插件转述任务，避免影响无关群聊。
- 主动私聊、休息判醒和跨群转述按 Host 任务/消息 ID 精确确认，热禁用或任务过期后不再误发送。
- 主动任务增加跨引用锚点防重和热重载短期墓碑，Host 回复 Bot 自己输出时不会形成第二轮重复发送。
- 主动任务元数据固定读取 Host 最后一条独立元数据行，契机文本中的伪标签或同名片段不能截断、遮蔽精确事件关联。
- 新入站会取消已生成但尚未交给适配器的旧回复；休息判醒按最终收口消息提交，不受 Planner 改选旧引用影响。
- 多图补话按合并后的完整图片组重新摘要；合并转发原文不会被本地规则误判成命令、叫醒或勿扰指令。
- 天气刷新串行化并校验查询城市，旧版无城市标记缓存及晚到的旧城市结果不会污染当前环境。
- 离线状态按日程边界补算睡眠、进餐和场景；关系结算按遗漏自然日追补。
- 群聊命令在数据库副作用前完成私聊权限检查；无效日程模板数组回退到内置生活骨架。
- 天气首刷改为后台任务，SnowLuma/NapCat 均继续使用 Host 标准消息契约。
- 配置 Schema 与插件版本升级为 `1.5.1`；SQLite Schema 升级为 v7，仅增加发送关联字段。

## [1.3.0] - 2026-07-13

### 新增

- RSS/Atom 新闻源、可读正文提取和可选 B 站插件 API 来源。
- 自建 SearXNG 与通用 JSON 主动搜索连接器。
- 新闻、搜索结果与麦麦人格、状态、能力和日程的自我关联。
- 近期新闻见闻与探索笔记命令，以及新闻/搜索/关联模型任务路由。

### 安全与兼容

- 所有联网能力默认关闭，不内置公共搜索节点或必须下载的依赖。
- 使用 Python 标准库 HTTP/XML/HTML，兼容 Windows/Linux 并遵循系统代理。
- 请求失败持久化退避，正文不可读时只保留来源摘要，不编造内容。
- 私聊话题默认不进入外部搜索词；搜索词本地清理账号、URL、邮箱与 @ 对象。
- 外部内容以不可信背景注入 Prompt 和 Planner，不执行来源中的指令。
- 配置 Schema 升级为 `1.3.0`，SQLite Schema 升级为 v4。
