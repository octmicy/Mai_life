# 麦麦生活（Mai_life）

`Mai_life` 让同一个麦麦拥有持续的生活状态、日程、睡眠、梦境和面向不同 QQ 用户的独立关系。当前版本为 **v1.7.2**，适配 MaiBot 1.0.12+、Plugin SDK 2.7.x，以及 SnowLuma、NapCat 两种 QQ 适配器。

## 快速配置

大部分设置直接在 MaiBot WebUI 的插件配置页完成，不建议手写完整 TOML。

1. 在“私聊用户”中填写真实 QQ 号，并启用该用户。
2. 最多设置一个 `owner`，其他用户使用 `friend`。
3. 为每个用户明确填写每日主动上限。建议主人 `2`，朋友 `1`，填 `0` 表示禁止主动私聊。
4. 在“环境与天气”中填写城市。插件会自动解析坐标，不需要经纬度。
5. 保存配置并重载插件，使用 `/mai_status` 检查任务和模型状态。

输入 `/mai` 或 `/mai_help` 会返回本地生成的指令菜单图。菜单由 Pillow 即时渲染，不访问生图 API，也没有新增独立 Web 页面或 Web 服务；Pillow、中文字体或图片发送能力不可用时会自动返回纯文本菜单。

用户、管理员、关系权限和群聊目标都只按 QQ 号判断。昵称与群名称由 MaiBot 会话自动读取，仅用于状态展示；改名不会改变身份或权限。

### 推荐开关

| 功能 | 默认 | 建议 |
| --- | --- | --- |
| 被动回复增强 | 开 | 保持开启 |
| 私聊消息防抖 | 开 | 保持默认等待 |
| 群聊消息防抖 | 关 | 确认群回复节奏后再开启 |
| 撤回增强 | 开 | 适配器需转发 OneBot 撤回通知 |
| 撤回摘要缓存 | 关 | 仅确有需要时开启 |
| 休息回复闸门 | 关 | 日程稳定后再开启 |
| 新闻阅读 | 关 | 配置搜索 API 后再开启 |
| 主动搜索 | 关 | 配置搜索 API 后再开启 |
| 社交转述 | 关 | 配置真实 QQ 群号后再开启 |
| 书柜与创作 | 关 | 了解 SQLite 明文存储风险后再开启 |

### 手动编辑规范

必须手动修改 `config.toml` 时：字符串和 QQ 号使用英文双引号，布尔值使用小写 `true/false`，时间使用 `HH:MM`，不要填写 `null` 或 `None`。`plugin.config_version` 由插件维护，不要修改。

## 联网搜索

新闻和主动搜索共用一条有序服务链。先开启“联网见闻”，再开启“新闻阅读”或“主动搜索”，最后在“联网搜索服务”中添加至少一个服务。

| 服务类型 | 需要填写 | 说明 |
| --- | --- | --- |
| 博查 `bocha` | 一个或多个 Key | 通常更适合中国大陆服务器 |
| Tavily `tavily` | 一个或多个 Key | 海外网络可能超时 |
| You.com `you` | 一个或多个 Key | 海外网络可能超时 |
| Responses `openai_responses` | Key、地址、模型名 | 使用带 `web_search` 工具的兼容接口，可接支持该协议的中转 |
| Chat `openai_chat` | Key、地址、模型名 | 模型本身必须具有联网能力，插件无法强制普通模型联网 |

自定义地址可以填写 API 基础地址，也可以填写完整的 `/responses` 或 `/chat/completions` 地址。插件不内置公共 Key、代理或中转地址。

每个服务可按优先级填写多个 Key，第一个为主 Key：

- 鉴权失败会停用该 Key，直到配置再次更新；
- `429` 会按 `Retry-After` 或指数时间冷却，再尝试备用 Key；
- 明确额度耗尽且没有重置时间时默认冷却 24 小时；
- DNS、超时、网络和 `5xx` 不会逐个消耗同服务的所有 Key，而是切换到下一个服务，并从 15 分钟开始退避；
- 单次逻辑搜索最多发起 12 次外部请求。

原始 Key 只保存在本地 `config.toml`。SQLite、日志、Prompt、Token 明细和管理输出只使用不可逆指纹，但服务器文件权限不当时 TOML 仍可能泄露 Key。

新闻默认每天最多读取 1 次，每次轮换一个普通兴趣词，最多保留 5 条结果并尝试读取前 3 篇正文。主动搜索默认每天最多 1 次。每日次数按整条降级链的一次逻辑尝试计算，失败或空结果也会计入，避免持续请求；所有服务失败时保留旧缓存，不生成虚假见闻。

v1.7.0 不再调用旧 RSS、Atom、B站插件 API、SearXNG 或通用 JSON 映射配置。升级后需要在 WebUI 添加新的搜索服务。

## 主要功能

### 生活与回复

- 全局维护精力、饥饿、心情、健康、睡眠、梦境、位置感和当前细化场景；不同用户看到的是同一条生活时间线。
- 每日生成生活框架，并在临近节点时细化场景、状态变化和主动分享契机。
- 时间、天气、节假日、农历和日程作为独立背景，不强迫 Replyer 逐项播报。
- 配置私聊可注入关系角色、当前状态、未完话题、图片摘要、未回应摘要和生活记忆。
- 主人与朋友使用不同隐私边界；朋友不会获得主人专属称呼、私人日记和敏感上下文。

### 消息体验

- 私聊默认按文本 1.2 秒、单图 3 秒、合并转发 2 秒收口，整轮最长 6 秒。
- 群聊防抖默认关闭；开启后使用独立的 0.8/2/1.2 秒参数，整轮最长 4 秒。
- 群轮次按“平台 + 群 QQ 号 + 发送者 QQ 号”隔离，不会合并不同群或不同用户。
- 配置私聊以及已开启防抖的群轮次中，同一最终消息只允许一次独立 Replyer 回复；正常分段不受影响。
- 撤回通知可取消尚未发送的回复和后续分段；插件不会撤回已经发送的 Bot 消息。
- 疑难图片支持单图、引用图、合并转发图和 GIF 抽帧摘要，失败或超时会回退到 MaiBot 原生多模态。

### 主动与社交

- 每个 QQ 用户拥有独立的关系温度、活跃时段、免打扰和明确每日主动额度。
- 主动契机先经过规则筛选，再交给 Planner 决定是否开口；Planner 沉默不消耗实际发送额度。
- 群聊观察只保存匿名短摘要，不保存群聊原文。
- `/mai_relay 群QQ号 内容` 只触发目标群 Planner，不构造 `@`，也不解析群名、昵称或别名。
- 群转私仅比较真实 QQ 号，并同时受离群时间、用户主动额度和隐私规则限制。

### 记忆与创作

- 梦境碎片、抽象日记和重要日期可以影响后续状态、日程与主动话题。
- v1.7.0 已删除技能成长；MaiBot 不是 Agent，插件不再模拟技能学习。
- 可选书柜保存麦麦自己的日记和作品。正文以 SQLite 明文保存，朋友只能访问允许公开的内容。

## 运行影响

| 项目 | 影响 |
| --- | --- |
| 响应延迟 | 私聊防抖固定增加最多约 1.2 秒；单图默认等待 3 秒。群防抖开启后按群参数增加等待。视觉和模型判醒还可能增加额外网络延迟。 |
| CPU 与内存 | 常规状态推进和 SQLite 操作较轻；首次打开指令菜单会在本地短时渲染 PNG，随后复用小型缓存。GIF 抽帧、图片摘要、群话题整理和正文清洗也会短时增加占用。 |
| Token | 日程、场景、梦境、连续话题和回复上下文会增加模型调用或 Prompt 长度；视觉、新闻批量整理、自我关联和自定义联网模型会进一步消耗 Token。 |
| 搜索 API 与费用 | 默认每天最多 1 次新闻搜索和 1 次主动搜索。主备 Key 和跨服务降级可能产生额外请求及费用，具体以服务商计费为准。 |
| 网络 | 博查通常更适合中国大陆网络；Tavily、You.com 和海外中转可能受 DNS、代理、防火墙或出口质量影响。所有联网模块均可完全关闭。 |
| 存储 | SQLite 保存生活、关系、摘要、作品、Key 健康指纹和调用统计；不保存搜索原始 Key 和图片二进制。TOML 中的 Key 与书柜明文需要限制文件权限。 |
| 命中率 | 主动发言和联网见闻取决于日程、兴趣、网络、模型、自我关联阈值和 Planner 决策，不承诺固定百分比或固定次数。 |

## 实现原理

规则层维护数值边界、日程校验、休息窗口、主动额度、Key 冷却和隐私边界；LLM 只负责叙事生成、摘要、查询规划和相关性判断。主动消息与跨群转述统一通过 `ctx.maisaka.proactive.trigger()` 交给 Planner 终审，插件不直接强制发送聊天内容。

SQLite Schema v9 使用事务和异步锁。升级会删除旧技能表、技能结算字段和手动关系别名数据，并把旧 `daily_proactive_max = -1` 转换为主人 `2`、朋友 `1`；其他生活、关系、新闻、日记和作品数据继续保留。数据库损坏或版本不兼容时会先保留原文件，再建立新库。

适配器兼容层只读取 MaiBot 标准消息结构，并归一化 SnowLuma/NapCat 的媒介和撤回通知。插件运行时不调用浏览器、Shell、Playwright、curl、ffmpeg 或其他外部二进制。

公开 API 保持：

- `maibot-community.mai-life.get_life_state`
- `maibot-community.mai-life.get_current_scene`
- `maibot-community.mai-life.get_schedule`
- `maibot-community.mai-life.get_user_relationship`
- `maibot-community.mai-life.get_environment_snapshot`
- `maibot-community.mai-life.create_proactive_opportunity`

私密话题、图片摘要、撤回摘要、搜索 Key 状态和 Token 明细不会通过无鉴权公共 API 暴露。

## 常见问题

**WebUI 保存提示 `NoneType`**
不要在 TOML 中填写 `null` 或 `None`。列表项未填写时使用空字符串、空列表或直接删除该未完成项。

**搜索一直降级**
使用 `/mai_admin sources` 查看服务类型、Key 指纹、状态和最近错误类别。先检查服务器是否能访问对应官方端点，再检查 Key、额度、自定义地址和模型名。

**群聊仍有连续回复**
群防抖默认关闭。开启后检查适配器是否提供真实群 QQ 号与发送者 QQ 号；该功能只合并同一用户在同一群中的连续补话。

**撤回后仍发出了一段**
适配器必须及时把 OneBot 撤回通知转发给 Host。已经完成平台发送的内容不会被插件追回，插件只取消尚未发送的部分。

## 开发测试

```powershell
$env:PYTHONPATH='D:\MaiBot\plugin\maibot-plugin-sdk-2.7.0;D:\MaiBot\plugin'
python -m unittest discover -s Mai_life\tests -v
python -m compileall Mai_life
```

可选依赖 `chinese-calendar`、`lunar-python` 和 `Pillow` 均可离线降级；未安装 Pillow 或系统缺少可用中文字体时，GIF 抽帧和图片指令菜单会分别使用各自的降级路径，不会阻止插件加载。
