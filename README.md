# 麦麦生活（Mai_life）

`Mai_life` 让同一个麦麦拥有持续的生活状态、日程、睡眠、梦境和面向多个私聊网友的独立关系。当前版本为 **v1.3.0**，适配 MaiBot 1.0.x 与 Plugin SDK 2.7.x。

QQ 接入同时支持 MaiBot 的 **SnowLuma Adapter** 与 **NapCat Adapter**。插件只读取 Host 标准化消息，并在兼容层统一两种适配器的引用、图片、语音、文件和合并转发结构。

## 快速配置

推荐只在 **MaiBot WebUI → 插件管理 → 麦麦生活 → 配置** 中修改。

首次使用主要完成四件事：

1. 在“私聊用户”中填写 QQ 号并启用档案；
2. 最多把一个用户设为 `owner`，其余保持 `friend`；
3. 在“环境与天气”中填写城市名，时区通常保持 `Asia/Shanghai`；
4. 检查“模型与成本编排”中的任务名与 MaiBot 模型配置一致。

默认任务名：

| 用途 | MaiBot 任务名 |
|---|---|
| 快速整理 | `utils` |
| 推理与日程 | `planner` |
| 梦境与叙事 | `replyer` |
| 图片理解 | `vlm` |

这里填写的是 MaiBot `model_task_config` 的**任务名**，不是模型名称、Provider ID 或 API Key。若没有配置 `vlm`，图片预摘要会自动降级，原生图片回复仍由 MaiBot 决定。

### 推荐开关

- 被动回复增强：开启；
- 私聊消息收口：开启；
- 同轮回复防重：开启；
- 休息回复闸门：默认关闭，确认普通聊天稳定后再开启；
- 图片转述增强：开启，没有可用 VLM 时自动降级；
- Token 监控：开启，只统计、不限制额度。
- 联网见闻、新闻阅读、主动搜索：默认全部关闭，配置可用来源后再开启。

### 手动编辑 TOML

WebUI 无法使用时才建议编辑 `config.toml`：

- 字符串和 QQ 号使用英文双引号；
- 布尔值只能写 `true` / `false`；
- 时间使用 `HH:MM`；
- 不要写 `null`、`None` 或不完整的配置项；
- `plugin.config_version` 必须存在；
- 最多只能配置一个启用状态的 `owner`。

## 功能

### 持续生活

全局维护精力、饥饿、心情、轻度健康、睡眠、梦境、位置感和当前活动。所有用户看到的是同一条生活时间线，但关系温度、主动额度、活跃时间和免打扰设置相互隔离。

每天生成粗粒度生活框架，在临近节点时细化成具体场景。核心数值由规则推进，LLM 只负责日程、场景和梦境叙事。

天气使用 Open-Meteo，用户只需配置城市，不需要经纬度。网络失败时使用缓存；没有缓存就显示“天气未知”，不会编造。

### 生活记忆

- 有效夜间睡眠后保存梦境摘要和少量醒后碎片；
- 每天整理一篇抽象日记，只读取麦麦的生活节点、梦境和互动数量；
- 明确的重要日期自动记录，模糊日期等待用户确认；
- 技能只能根据日程实践等真实证据缓慢增长，并受每日上限约束。

生活记忆默认开启。逐消息日期模型分析和技能模型分析默认关闭，可在 WebUI 的“生活记忆”中按需启用。朋友无法读取私人日记；每个用户只能管理自己的重要日期。日期影响全局日程时会先匿名化，不携带用户 ID 或自定义事件原文。

### 被动回复增强

仅向配置的私聊用户注入：

- 关系角色和关系温度；
- 当前生活状态与细化场景；
- 当前/下一日程；
- 当前消息意图和未完话题；
- 休息期间未回应摘要；
- 时间、工作日、农历节气、平台和消息媒介。

Planner 获得完整结构化背景，Replyer 只获得压缩摘要。背景不要求主动提及，天气和日程也不会被伪装成用户刚刚说过的内容。

连续话题只是轻量元数据，不替代 MaiBot 自身记忆，也不会复制完整聊天历史。

### 主人和朋友

- `owner`：最多一人，可延续主 MaiBot 人格中明确属于主人的专属关系；
- `friend`：普通朋友边界，不使用主人/恋人称呼，不注入主人专属或私密上下文。

朋友回复命中配置的主人专属词时会本地触发一次重生成；仍不合规则静默取消。建议主 MaiBot 人格也明确写出“专属称呼只对指定主人使用”。

用户级主动上限设置为 `-1` 时，主人默认每天 2 次、朋友默认每天 1 次；设为 `0` 可禁止该用户的主动消息。

### 消息收口与重复回复保护

配置私聊默认按以下节奏等待补充：

- 普通文本：1.2 秒；
- 单张图片无说明：3 秒；
- 合并转发：2 秒；
- 整轮最长：6 秒。

短时间连续补话会按顺序合并，只有最新 Hook 进入 MaiBot 主链。紧急、明确叫醒和明确勿扰消息会提前结束等待。

同轮回复防重以最终用户消息 ID 为锚点，只允许一次非空 Replyer 结果；同一次回复由 Host 正常拆成多段不受影响。这用于抑制 Planner 对同一消息重复调用 `reply`，不是简单的文本去重。

首版不处理群聊收口，避免多人聊天顺序被私聊参数打乱。

### 休息回复闸门

闸门默认关闭。开启后，必须同时满足：

1. 当前时间位于夜间 `22:30–08:00` 或午休 `12:00–14:30`；
2. 当前日程为 `sleep`、`nap` 或配置允许的 `rest`。

明确勿扰会静默，紧急和明确叫醒优先放行。普通消息可按概率或模型评分判断。

判定放行并不立即改变睡眠状态：只有平台确认第一段回复实际发送成功后，才切换为 `woken` 并开始醒后缓冲。Planner 沉默或发送失败不算醒来。

### 图片转述增强

插件只预摘要较难由普通消息链稳定处理的情况：

- 私聊单图且没有说明；
- 引用消息中的图片；
- 合并转发图片；
- 动态 GIF。

视觉摘要最多等待 6 秒，超时后直接交回 MaiBot 原生多模态。GIF 可使用 Pillow 抽取最多 4 帧，不依赖 ffmpeg；未安装 Pillow 时会降级。

数据库只保存图片哈希、短摘要和来源线索，默认 24 小时后过期，不保存图片二进制，也不会根据图片猜测人物真实身份。

### 环境与历法

时间、星期、时段、平台和消息媒介完全本地识别。以下纯 Python 包为可选增强：

```text
chinese-calendar
lunar-python
```

它们分别提供中国调休/节假日和农历/节气。未安装时插件仍可在 Windows、Linux 和无外网服务器运行，并降级为普通星期判断和“农历未知”。

### Token 监控

分别记录：

- Mai_life 主动发起的模型任务；
- 插件观察到的 Planner/Replyer Token。

统计包括任务名、模型名、输入/输出/总 Token、耗时和失败。当前版本不提供每日硬限额。

### 联网见闻

新闻支持 RSS、Atom、自建 RSSHub，以及其他插件公开的 B 站消息 API。正文可读时优先整理正文；请求失败或正文不可读时保留已有缓存和订阅摘要，不补写不存在的内容。

主动搜索支持自建 SearXNG 和通用 JSON 接口。插件不内置公共搜索节点，也不调用浏览器、Shell、curl 或外部二进制。新闻和搜索默认关闭，适合中国网络环境的做法是使用国内可访问订阅、自建 RSSHub/SearXNG，或保持完全离线。

外部内容会先判断与麦麦人格、状态、技能、创作和日程是否真正相关，达到阈值才形成主动契机。所有外部文本都标记为不可信背景；默认不会把私聊话题转成搜索词。

- `get_life_state`
- `get_current_scene`
- `get_schedule`
- `get_user_relationship`
- `get_environment_snapshot`

## 实现原理

入站顺序：

```text
消息识别 → 配置用户检查 → 私聊收口 → 疑难图片摘要
        → 互动记录 → 休息闸门 → Planner/Replyer
        → 朋友边界/轮次锁 → 平台实际发送确认
```

后台任务、视觉任务和连续话题任务都保存 `asyncio.Task` 引用；禁用、热更新和卸载时统一取消并等待。SQLite 使用异步锁和事务，schema 初始化可重复执行。损坏或未来版本数据库不会被静默覆盖。

主动聊天只创建结构化契机，并通过 `ctx.maisaka.proactive.trigger()` 交给 Planner 终审。每日额度只在平台实际发送成功后计数。

### 代码结构

```text
Mai_life/
├── plugin.py          # 插件入口、Hook、命令和公开 API
├── config.py          # WebUI 配置模型
├── core/              # SQLite、环境信息和模型调用
├── life/              # 状态、日程、休息、关系与主动行为
├── information/       # 新闻、搜索、正文解析与自我关联
├── messaging/         # 消息收口、提示词和图片理解
└── tests/             # 契约、核心逻辑和消息体验测试
```

## 常见问题

### WebUI 保存时报 TOML `NoneType`

不要填写 `null` 或 `None`。插件默认配置不包含不可序列化空值；需要留空的文本字段使用空字符串。

### 图片增强显示降级

检查 MaiBot 的 `model_task_config.vlm` 是否存在且绑定支持视觉的模型。没有 VLM 不会阻塞普通回复。

### 农历或节假日显示未知

这是可选依赖未安装或日期超出依赖覆盖范围，不会联网查询。可使用国内 PyPI 镜像安装可选包，也可以保持降级状态。

### 仍然出现重复回复

先确认“同轮回复防重”已开启，再检查日志中的用户消息 ID。该保护只合并同一消息锚点；新用户消息、不同引用目标和主动触发属于不同轮次。

### 撤回为什么不能取消回复

NapCat Adapter 已能上报部分 OneBot 撤回通知，但 SnowLuma Adapter 尚未提供一致、可靠的取消链路。为保证两个适配器行为一致，当前版本不实现只对单一适配器有效的猜测式撤回；待 Host/适配器形成统一通知契约后再加入。
