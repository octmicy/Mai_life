# 麦麦生活（Mai_life）

`Mai_life` 是面向 MaiBot Plugin SDK 2.7.x 的生活模拟插件。

当前独立仓库版本：**v1.0.0**。插件版本与 `plugin.config_version` 是两个概念：前者表示发布版本，后者表示配置结构版本。

插件让麦麦拥有一条持续运行的生活时间线，包括精力、饥饿、心情、睡眠、梦境、天气、日程和当前位置。麦麦的生活状态在所有配置用户之间共享，但每个用户的关系温度、活跃度、免打扰时间和主动消息额度相互独立。

## 主要功能

- **生活状态**：维护精力、饥饿、心情、轻度健康、睡眠和当前位置感。
- **天气背景**：根据配置城市自动查询 Open-Meteo 天气，无需填写经纬度。
- **每日生活框架**：每天生成睡眠、吃饭、学习、工作、出行和休闲安排。
- **临近场景细化**：日程临近时再生成具体场景、状态变化和分享契机。
- **梦境系统**：有效夜间睡眠后生成梦境碎片，并留下轻微情绪余韵。
- **休息回复闸门**：睡眠或午休时，可判断是否被普通私聊消息叫醒。
- **多用户关系**：每位用户独立维护关系温度和互动记录。
- **自然主动私聊**：结合日程、精力、心情、关系和用户活跃时间选择主动契机。
- **Planner 终审**：插件只提供主动候选，最终是否发送及发送内容由 MaiSaka Planner 决定。

# 快速开始

## 1. 安装

将 `Mai_life` 放入 MaiBot 的插件目录，然后重新加载插件或重启 MaiBot。

```text
MaiBot/
└─ plugins/
   └─ Mai_life/
```

## 2. 在 WebUI 中配置

推荐全部通过 MaiBot WebUI 设置，不需要手动编辑 `config.toml`。

首次使用主要修改以下内容：

### 私聊用户

在“私聊用户”中添加你的 QQ：

- QQ 号：填写实际 QQ 号；
- 启用用户档案：开启；
- 允许主动私聊：按需要开启；
- 用户称呼：可留空；
- 初始关系温度：推荐 30；
- 免打扰时间：默认 `00:00～08:00`。

目标用户需要至少与 MaiBot 私聊过一次，插件才能解析对应的会话 ID。

### 环境与天气

在“环境与天气”中设置：

- 时区：中国大陆通常使用 `Asia/Shanghai`；
- 天气城市：例如 `Shanghai`、`Beijing`、`Hangzhou`；
- 天气刷新间隔：推荐 30～60 分钟。

插件会自动把城市名转换为天气坐标，无需设置经纬度。

### 主动私聊

默认采用自然低频：

- 每位用户每天最多 2 次；
- 同一用户最小间隔 180 分钟；
- 用户刚说过话后等待 30 分钟；
- 麦麦精力低于 25 时不主动聊天。

建议先使用默认值运行一段时间，再根据实际体验调整。

### 休息回复闸门

休息闸门默认关闭。建议确认普通聊天稳定后再开启。

可选模式：

- `probability`：规则加概率判断，速度快，不增加模型调用；
- `llm`：使用模型判断是否应该醒来，理解更准确但会增加延迟和消耗。

## 3. 手动编辑配置时的注意事项

一般不需要手动修改 `config.toml`。如果确实需要：

- 不要修改 `plugin.config_version`；
- 字符串使用英文双引号；
- 布尔值使用小写 `true` 或 `false`；
- 时间使用 `HH:MM`；
- 不要填写 `null` 或 `None`；
- 多用户需要重复完整的 `[[users.profiles]]` 配置块；
- 修改后重新加载插件。

# 功能说明

## 生活状态

麦麦维护以下全局状态：

| 状态 | 说明 |
|---|---|
| 精力 | 0～100，清醒时消耗，睡眠时恢复 |
| 饥饿 | 0～100，数值越高越饿，进餐后降低 |
| 心情 | -1～1，受精力、饥饿、梦境和生活事件轻微影响 |
| 精神活跃度 | 0～1，睡眠时下降，清醒活动时恢复 |
| 健康 | 只模拟疲劳、饥饿等轻度不适 |
| 睡眠阶段 | 清醒、入睡、浅睡、深睡、被叫醒 |
| 当前位置 | 家里、卧室、厨房、街道等生活场景 |
| 当前活动 | 来自临近日程细化后的真实场景 |

状态由规则推进，LLM 不会直接修改核心数值。插件重启后会根据最后更新时间补算一部分离线状态。

## 天气

天气只作为独立环境背景参与判断，不会直接覆盖麦麦当前正在做的事。

处理流程：

```text
配置城市
→ 自动解析城市坐标
→ 获取 Open-Meteo 天气
→ 写入本地缓存
→ 被动回复只读取缓存
```

网络失败时使用最近缓存；没有缓存时显示“天气未知”，不会编造天气。

## 日程与场景

插件每天生成粗粒度生活框架，节点类型包括：

- `sleep`：夜间睡眠；
- `nap`：午休；
- `meal`：吃饭；
- `work`：工作；
- `study`：学习；
- `travel`：出行；
- `leisure`：休闲；
- `rest`：普通休息。

日程临近时，插件会细化当前段和下一段，生成：

- 具体生活场景；
- 精力、饥饿和心情变化；
- 是否适合分享；
- 主动分享主题和权重；
- 契机过期时间。

LLM 返回的日程会经过格式、时间、重叠、睡眠和进餐校验。生成失败时使用 `mai_template.json` 降级。

## 睡眠与梦境

麦麦进入睡眠日程后会逐步进入浅睡或深睡。有效夜间睡眠结束后，插件会生成一段简短梦境。

梦境：

- 不会被当作预言；
- 不会强制在每次聊天中提起；
- 只对心情和精力产生轻微影响；
- 可以成为一个低权重的主动分享契机。

## 休息回复闸门

启用后，仅对已配置用户的私聊生效。

判断顺序：

```text
是否处于睡眠或午休日程
→ 是否仍在醒来缓冲期
→ 明确勿扰则保持睡眠
→ 明确叫醒或紧急情况直接醒来
→ 普通消息按概率或 LLM 判断
```

未回复消息只保存简短摘要，不会在自然醒后自动补发；下一次正常对话时可作为背景使用。

## 关系温度

每位用户独立维护 0～100 的关系温度：

| 分数 | 阶段 |
|---:|---|
| 0～19 | 陌生 |
| 20～39 | 认识 |
| 40～59 | 熟悉 |
| 60～79 | 亲近 |
| 80～100 | 知己 |

关系温度会根据有效互动、消息数量和主动消息回应缓慢变化。它只影响熟悉程度和分享意愿，不代表自动进入恋爱关系。

## 主动私聊

主动私聊首先执行硬性过滤：

- 插件或用户未开启主动功能；
- 麦麦正在睡眠；
- 用户处于免打扰时段；
- 达到每日上限；
- 未达到最小间隔；
- 用户刚刚发过消息；
- 麦麦精力过低；
- 没有有效生活契机。

通过过滤后再综合评分：

- 当前生活契机；
- 精力和心情；
- 用户关系温度；
- 用户常见活跃时间；
- 距离上次互动的时间；
- 是否存在未回应消息摘要。

同一个生活契机最多选择一个用户，避免把同一件事复制发给所有人。

# 插件原理

## 运行流程

```text
状态维护循环
├─ 刷新天气缓存
├─ 确保当日日程存在
├─ 细化临近生活场景
├─ 推进生活数值
├─ 应用已结束场景的状态变化
└─ 结算用户关系温度

主动巡检循环
├─ 检查用户额度和免打扰
├─ 检查麦麦睡眠、精力和日程
├─ 获取未过期生活契机
├─ 对用户和契机进行评分
├─ 原子消费一个契机
└─ 交给 MaiSaka Planner 最终决定
```

## Prompt 分层

插件不会把所有信息混成一段强制人设，而是分为：

```text
【麦麦内在生活状态】
精力、饥饿、心情、健康、睡眠、梦境、当前位置和当前场景

【独立环境背景】
时间、天气、当前日程和下一日程

【当前网友关系】
关系温度、关系阶段和未回应消息摘要
```

Planner 获取完整背景进行决策，Replyer 获取压缩摘要保持表达一致。Prompt 会要求模型在信息无关时不要强行提及天气、日程或状态数值。

## SQLite 数据

运行数据保存在：

```text
Mai_life/data/mai_life.db
```

主要数据表包括：

- 全局生活状态；
- 睡眠运行状态；
- 梦境；
- 每日日程；
- 细化场景；
- 主动契机；
- 用户关系；
- 互动记录；
- 主动发送记录；
- 未回应消息；
- 天气缓存。

涉及多项状态的写入使用 SQLite 事务。数据库损坏时会保留原文件，不会直接静默覆盖。

## 后台任务与热更新

插件维护：

- 状态维护任务；
- 主动巡检任务；
- 每日日程生成任务。

配置热更新和插件卸载时会取消并等待旧任务结束，避免多次热更新后出现重复调度循环。

## 主动发送确认

调用 `proactive.trigger` 不代表 Planner 一定发送消息。插件会先创建 pending 记录，只有 Replyer 在确认窗口内产生实际回复后，才增加每日主动发送次数。

普通被动回复没有 pending 记录，不会创建无意义的 SQLite 写事务。

# 插件 API

- `maibot-community.mai-life.get_life_state`
- `maibot-community.mai-life.get_current_scene`
- `maibot-community.mai-life.get_schedule`
- `maibot-community.mai-life.get_user_relationship`
- `maibot-community.mai-life.create_proactive_opportunity`

创建外部主动契机示例：

```python
await ctx.api.call(
    "maibot-community.mai-life.create_proactive_opportunity",
    topic="刚看完一部有意思的电影",
    motive="想找一个熟悉的网友分享",
    weight=0.6,
    expires_minutes=120,
)
```

# 常见问题

## WebUI 保存失败

- 重新加载插件并刷新页面；
- 不要在配置中填写 `null` 或 `None`；
- 确认配置版本为插件自动生成的最新版本；
- 手动修改配置后检查 TOML 格式。

## 天气显示未知

- 检查天气城市是否为空；
- 尝试常见英文城市名；
- 检查服务器网络；
- 查看 `[MaiLife] 天气刷新失败` 日志。

## 麦麦不主动聊天

- 检查用户档案和主动私聊开关；
- 确认目标用户与 MaiBot 私聊过；
- 检查免打扰、每日上限和最小间隔；
- 检查麦麦是否正在睡眠或精力过低；
- Planner 可能根据上下文选择沉默。

## 出现重复回复

如果同一条用户消息对应多个不同的 `reply` 工具调用，通常是 MaiSaka Planner 多轮决策造成，而不是插件主动系统重复触发。

插件主动触发时会出现日志：

```text
[MaiLife] 主动候选交给 Planner
```

## 数据隐私

数据库可能包含 QQ 用户 ID、互动时间、简短消息摘要和关系状态。不要公开发布带有真实运行数据的 `data/` 目录。

# 开发与测试

运行测试：

```bash
python -m unittest discover -s Mai_life/tests -v
```

当前测试覆盖配置 Schema、SQLite、生活状态、日程校验、休息闸门、关系隔离和主动发送确认。

`Mai_life` 的目标不是让麦麦反复汇报状态，而是让生活背景在长期运行中保持连续。当这些背景与当前话题无关时，麦麦应该像普通网友一样自然聊天。
